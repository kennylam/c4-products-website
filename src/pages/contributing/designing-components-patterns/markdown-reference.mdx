## Emphasis

### Italics
*Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.*

The following code renders text in *italics* shown above:
```
*Lorem ipsum dolor sit amet, consectetur adipiscing elit ...*
```

```
*these_words_are_italic*
```

### Bold
**Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.**

The following code renders text in **boldface** shown above:
```
**Lorem ipsum dolor sit amet, consectetur adipiscing elit ...**
```

```
**these_words_are_bold**
```

### Bold and italics together
***Lorem ipsum dolor sit amet, consectetur adipiscing elit, sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.***

The following code renders text in ***italics and boldface*** shown above:
```
***Lorem ipsum dolor sit amet, consectetur adipiscing elit ...***
```

```
***these_words_are_italic_and_bold***
```

### Linebreaks

Lorem ipsum dolor sit amet, consectetur adipiscing elit, <br />
sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.

The following code adds a linebreak/new paragrpah shown above:
```
Lorem ipsum dolor sit amet, consectetur adipiscing elit, <br />
sed do eiusmod tempor incididunt ut labore et dolore magna aliqua.
```

```
<br />
```

Note: Be sure to use the self-closing syntax for the &lt;br /> tag. Don't just write &lt;br> or you'll get an error 
(the parser will complain that the &lt;br> tag has been left unclosed) and the post will not be published.

## Block Quotes
A block quote is a piece of text that is *indented* from the rest of the text. Examples are seen in poetry or an email:

> Dear Arvind
<br />
>
> *"The trouble with every one of us is that we don't think enough. We don't get paid for working with our feet — we get paid for working with our heads"*
<br />
>
> Best,
>> Drew Glapa

The following code renders the block quotes shown above:
```
> Dear Arvind
<br />
>
> *"The trouble with every one of us is that we don't think enough. We don't get paid for working with our feet — we get paid for working with our heads"*
<br />
>
> Best,
>> Drew Glapa
```

```
> One layer of indentation
>> Two layers of indentation
>>> Three layers of indentation
```
Note: Each carot adds a new layer of indentation

## Lists
There are several ways to organize related items into unordered, ordered & nested lists in Markdown.

### Unordered lists

* Oranges
* Apples
* Berries

The following code renders the undordered list shown above:
```
* Oranges
* Apples
* Berries
```

```
* Item 1
* Item 2
* Item 3
```
An asterisk (\*), minus sign (\-) or plus sign (\+) can be used for unordered lists

### Ordered lists

1. Oranges
2. Apples
3. Berries

The following code renders the ordered list shown above:
```
1. Oranges
2. Apples
3. Berries
```

```
1. Item 1
2. Item 2
3. Item 3
```

### Nested lists

* Apples
* Oranges
* Berries
  * Strawberries
  * Raspberries

The following code renders the nested list shown above:
```
* Apples
* Oranges
* Berries
  * Strawberries
  * Raspberries
```

```
* Item 1
* Item 2
* Item 3
  * Item 3.1
  * Item 3.2
```

## Escaping characters
Certain characters, such as the hash symbol (\#) and the plus sign (\+), can be interpreted by Markdown as part of its syntax. 
To print one of these characters literally on screen, you can escape it using a backslash (\\). **Ex.** 
"\\#" prints the "\#" character itself, instead of creating a <a href="usage#headings">heading</a>. <br />

\# Not a heading

The following code renders an actually '\#' that is not a heading shown above:
```
\# Not a heading
```

The following characters can be escaped using a backslash (\\) to ensure they are rendered literally on screen: 

| Symbol | Name |
| --- | --- |
| \* | asterisk |
| \\ |	backslash |
| \`	| backtick |
| \[\]	| brackets |
| \{\} | curly braces |
| \(\) |	parentheses |
| \. |	period or dot |
| \# |	hash or pound sign |
| \+ |	plus sign |
| \-	| minus sign or hyphen |
| \_	| underscore |
| \! |	exclamation mark |
| \| |	pipe |
