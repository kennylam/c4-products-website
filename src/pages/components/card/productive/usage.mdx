## Overview

Productive cards include more content than expressive cards, offering an overview of often dynamic information that the user can manipulate or expand in various ways.


### When to use

<Row>
<Column colMd={8} colLg={8}>

Productive cards are largely interactive grouping of information and are often used in homepages, dashboards, or reporting interfaces.

</Column>

</Row>

<Row>
<Column colMd={8} colLg={10}>

- Use a productive card when the data within needs to be manipulated or filtered.
- Use a productive card when the container can resize or reorder.
- Use productive cards to present digestible summaries of dynamic information.


</Column>
</Row>

### When not to use

<Row>
<Column colMd={8} colLg={10}>

- Do not use productive cards to group multi-step tasks. Use the card to guide the user to a new page, panel, modal, or tearsheet to complete a complex task.
- Consider using a [tile](https://www.carbondesignsystem.com/components/tile/usage) if the content cannot be summarized or related to more than one section or topic. 
- Do not use productive cards for large calls to action. Use an [expressive card](https://pages.github.ibm.com/cdai-design/pal/components/card/expressive/usage) instead.  


</Column>
</Row>

## Formatting
### Anatomy

<Column colMd={8} colLg={8}>

#### 1. Action title block
Contains the header text and label or description of the card's purpose. The block's right side is the primary toolbar for actions that manipulate the card content. 

#### 2. Body content
The body content area displays various elements based on the use-case. Productive card content is dynamic and often can be editable.

#### 3. Supplemental action bar (optional)
The secondary action bar provides the user with direct access to a new page that expands the productive card’s information. 

</Column>

<Aside>
When the card uses a label only instead of a title, the card uses a complex bottom action bar. See action bar section for more guidance.   
</Aside>

</Column>
</Row>

### Sizing
Both the expressive and the productive card have four widths available for use based on the Carbon 16 column grid.
- Use small as the standard size for most of the expressive cards and productive cards.
- Use medium for more essential callouts. Large size works well for productive cards with bigger charts.
- Use large and full width for productive cards when there are multiple levels of information hierarchy and complex charts.

#### Complex bottom action bar
Use a complex bottom bar for both primary and supplemental action **only** when: 
- The card doesn’t have a title and only uses a label in the title block.
- The title is longer than two lines and because of its relevance, it can't be truncated.
- There are multiple action states.

Do not use complex bottom action bars for chart actions, nor in combination with the standard primary action bar. If used, keep consistency across the page layout.

Place all manipulative actions to the right and supplemental actions to the left of the bar. 

</Column>

## Content
### Main elements

<Title>Title (optional)</Title>

- The title should clearly describe the purpose of the card. 
- The title should be brief and truncated after two lines.
- You can use an optional label above the title to set the context for the panel’s information or a description below to expand the title’s content. 
- Use a label if the card does not have a title.


<Title>Body copy</Title>

- The content type can vary depending on the card’s purpose; it often compiles a set of detailed information relevant to the user’s workflow.  
 - Content can include text, tables, charts, lists, or status icons that can be manipulated by the user.
- The content usually has multiple information hierarchy levels, so the composition should respond to the use case.


<Title>Charts</Title>

- Charts are often used within the body content block, especially on dashboard or reporting interfaces. Follow the Carbon guidance on charts.


<Title>Primary and ghost buttons</Title>


- Primary actions can include editing, creating, filtering, removing or deleting, configuration, etc.
- Supplemental action takes the user to a new page and communicates what the user will see.
- Keep actions brief and relevant to the use case. If the number of primary actions surpasses the number allowed per card’s size, use an overflow menu.
- Use descriptive words for the actions. Avoid vague words.  
- Supplemental actions should be minimal and on a lower hierarchy than manipulative actions. 
- Card using primary and supplemental actions should follow the Carbon guidelines for [button usage](https://www.carbondesignsystem.com/components/button/usage) and action labels.

## Best practices

### Do’s

- Use standard action bars as the default unless the card meets the cases to use a complex bottom bar.
- Use a limited number of internal links.
- Use a 16px or 2px gutter between cards in page layouts.
- Truncate the title at two lines. 


### Don’ts

- Do not mix standard action bars with complex bottom bars in the same layout.
- Do not use more than four internal links inside a card. 
- Do not use clickable cards when there is an internal link. 
- Do not use primary buttons in productive cards.
