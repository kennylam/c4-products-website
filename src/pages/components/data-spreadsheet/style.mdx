## Structure

### Headers

**Column** headers can change height independantly if designed. For example, if a 64px tall header height is required for 2 (two) lines of text (title and caption), then the rows may be any height desired. The default is to match both heights.

**Row** headers must match the row height.

<Column colMd={8} colLg={8}>

![Data spreadsheet](images/style/frame3.png)
![Data spreadsheet](images/style/frame4.png)

</Column>

### Cells

Cells share similar structure as headers except it is discouraged to use buttons in cells. There can be exceptions, but a rule of thumb will be to avoid buttons in cells because it does not fit for extra small (24px) tall cells.

<Column colMd={8} colLg={8}>

![Data spreadsheet](images/style/frame5.png)

</Column>

### Columns

To lead the eye down a spreadsheet column with ease, do not add visual obstacles unless intended to disrupt the flow. Keep all content aligned as much as possible.

| Spacing between       | Property                | px / rem                  | Spacing token         |
| :--                   | :--                     | :--                       | :--                   |
| Columns               | Padding                 | 16 / 1                    | `$spacing-05`         |

### Enhancements

Most enhancements live in headers but some may be adapted to live inside cells, such as icons.

| Elenment                    | Property                   | px / rem     | Placement         |
| :--                         | :--                        | :--          | :--               |
| Checkbox                    | height, width              | 20 / 1.25    | Header            |
| Radio button                | height, width              | 20 / 1.25    | Header            |
| Overflow menu               | height                     | 40 / 2.5     | Header            |
| Icon                        | svg                        | 16 / 1       | Header, cell      |

### Headers


<Accordion>

<AccordionItem title="Buttons">
<Row>
<Column colMd={8} colLg={8}>

Alignment: right (columns), left (rows)

- Overflow menu buttons are typically the ones used in headers. These are right aligned.
- Using the same location for buttons in the headers (i.e keeping the button right aligned) helps navigating the headers, making it expected to where a user should click.
- In the event that a header requires an icon or symbol to be right aligned, make the button left aligned (rare).
- Use 8px spacing between the button and the text.
- Overflow menu buttons should be persistently visible.
- Rarely are they used in rows, but may be placed if necessary.
