<PageDescription>

  Providing a consistent Carbon for Cloud &amp; Cognitive approach to using certain Carbon assets

</PageDescription>

There is a degree of variation possible within some of the Carbon components and patterns. As we want 
the Cloud &amp; Cognitive portfolio to have a high degree of visual and interaction consistency, we have aligned 
on the following options for all our offerings:

[Carbon data table component](https://www.carbondesignsystem.com/components/data-table/usage/)
<ul style="margin-top: 0">
  <li>We do not use zebra striping</li>
  <li>We always dispaly the overflow menu</li>
</ul>

[Carbon notification component](https://www.carbondesignsystem.com/components/notification/style/)
<ul style="margin-top: 0">
  <li>We use the low contrast style</li>
</ul>

[Carbon UI shell](https://www.carbondesignsystem.com/components/UI-shell-header/usage)
<ul style="margin-top: 0">
  <li>Our header and left-side rail use the dark theme</li>
</ul>

[Background colour](https://www.carbondesignsystem.com/guidelines/color/usage/)
<ul style="margin-top: 0">
  <li>We generally use Gray 10 as the background color on all light theme pages (although there are some valid exceptions)</li>
  <li>We generally use Gray 100 as the background color on all dark theme pages</li>
</ul>
