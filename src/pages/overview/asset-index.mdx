#### Asset

* The Asset column provides a link to the asset specification, guidelines, or in-progress GitHub issue (tagged with `GHE issue`).
* All components and patterns displayed on this asset index are available to be used by everyone. Note: Some assets may come from different code respositories. Before using an asset, review the list of <a href="https://pages.github.ibm.com/cdai-design/pal/overview/carbon-specifications#carbon-for-ibm-products-core-assets" target="_blank">core assets for IBM Products<Launch16 fill="#0F62FE" style="vertical-align:middle; margin: 0 2px 2px 0"/></a> and <a href="https://pages.github.ibm.com/cdai-design/cloud-pal/cloudpal-guidance/about/adopting-cloud-pal#core-cloud-pal-component-usage-and-ui-requirements" target="_blank">core assets for IBM Cloud<Launch16 fill="#0F62FE" style="vertical-align:middle; margin: 0 2px 2px 0"/></a>.
* Core assets that the majority of IBM Products are expected to adopt are indicated with a <Tag type="green">IBM Products Core Asset</Tag> tag. 
* Core assets that the majority of Cloud PAL products are expected to adopt are indicated with a <Tag type="cyan">Cloud PAL Core Asset</Tag> tag. 

#### Contributors

* The points of contact for information about the asset.

#### Design status

* `In progress`: a workgroup is currently working on the design specification.
* `Done`: the asset has been designed, evaluated, and approved by the DSAG / DSIT.
* When an asset has been approved and published on this site or on the [Carbon community assets](https://www.carbondesignsystem.com/community/overview/) page),
  code implementation can begin.
* Our design assets that are available within the [Carbon community assets](https://www.carbondesignsystem.com/community/overview/) area are tagged with `Carbon`.


#### Respository
Assets on Carbon for IBM Products can be found on two seperate repositories. 
* `CDAI-design/pal` The [Carbon for IBM Products library](https://github.com/carbon-design-system/ibm-cloud-cognitive)
* `ibmcloud/pal` [The Cloud PAL library](https://github.ibm.com/ibmcloud/pal)


#### Tags
* `GA`: This asset is available with full functionality and documentation.
* `NA`: Code implementation work not yet possible as the design is still in progress.
* `Needs dev resource`: The design is done and we're now waiting for a development resource to get this asset implementd in code. If you think you can help, see our <Link to="/contributing/coding-components-patterns/overview">Coding components & patterns</Link> page.
* `GHE Issue`: denotes that the issue's progress exists inside an issue on the Github repo. 
* `Experimental`: These assets are ready to use but may lack parts of the documentation, or extra functionality is in development.
* `Available in Canary`: the component is coded and available but still going through final release review testing.
