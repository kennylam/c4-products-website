#### Status

* Status is used by maintainers to set consumption expectations of stability and the likelihood of future changes.
The `status` key canhave the following values:

<br/>

| Status | Description |
| ----------- | ----------- |
| `Not started` | Asset is in backlog, no active work has been started |
| `Draft` | A workgroup is currently working on the design specification |
| `Experimental` | These assets are ready to use but may lack parts of the documentation, or extra functionality is in development |
| `Stable` | The asset has been designed, evaluated, and approved by the DSAG / DSIT |
| `Deprecated` | These assets have been discontinued and no longer have active contribution |


#### Respository
* Assets on Carbon for IBM Products can be found on two seperate repositories

<br/>

| Name | Link |
| ----------- | ----------- |
| `CDAI-design/pal` | [Carbon for IBM Products library](https://github.com/carbon-design-system/ibm-products) |
| `ibmcloud/pal` | [Cloud PAL library](https://github.ibm.com/ibmcloud/pal) |
