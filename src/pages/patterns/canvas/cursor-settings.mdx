## Overview

The examples below are built from precedents that exist not only in IBM products but in canvas and drag-and-drop interactions that span across several types of experiences.

These cursor states mimic the following precedents: MacOS, diagrams.net, Informatica, Keynote, Lucidcharts, Microstation, Sketch, Talend, and many others.

## Default cursor state

Consistency is key when it comes to cursor states. The `default` state is the standard cursor style. In canvases, the cursor should remain that style unless otherwise noted.

## Panning the canvas

Using hotkeys (e.g. - spacebar) will allow the user to change the cursor state and access frequently-used secondary actions. 

In the example of panning around the canvas, the user will hold down the `spacebar` to engage the panning interaction. Clicking the mouse while in the grab state will change the cursor to the grabbing state, allowing the user to pan the canvas.

## Moving items on canvas

Similar to moving items in MacOS' Finder or Sketch, the state for moving items on the canvas will be the `default` cursor state. However, if the use case justifies it, an exception is allowed in the form of the `move` cursor state.

## Drag-selecting over canvas

Drag-selecting allows the user to select multiple canvas objects. Drag-selecting is accessed by clicking an empty area of the canvas and dragging to create a `region selector`, the box that identifies the selections extents.


<Row>
  <Column colMd={8} colLg={8}>

![Drag-selecting 4 nodes.](/images/drag-selectCursor.png)

<Caption>
  Drag-selecting 4 nodes.
</Caption>

  </Column>

When dragging to select, the region selector should use the following settings...


 |   Region selector property     |      Carbon token, additional CSS spec     |  
| ------------- | ----------- | 
| Background color     | `$ui-03`, opacity: 0.5 | 
| Border (1px)  | `$ui-05` |
